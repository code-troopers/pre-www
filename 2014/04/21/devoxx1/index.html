Retour de devoxx : Améliorer son live-coding

  • devoxx
  • Intellj
  • live-coding

Je suis un développeur passionnné1 et, lors des conférences, JUG (ou toutes autres présentations) j’aime bien quand le speaker présente un petit bout de code pour illustrer ses propos. Lorsque c’est du live coding, c’est encore mieux. Le seul problème c’est que quand on n’est pas au Mariott et qu’il n’y a pas un vidéoproj de malade, en général dans l’audience, passé le 3ème rang, plus personne n’y voit rien.

Avant la présentation 42 IntelliJ IDEA tips and tricks de Hadi Hariri (twitter) la solution c’était de fermer toutes les views autour de la classe ouverte puis de faire un ctrl+scroll pour zoomer. Cette époque est maintenant révolue, en effet, IntellJ possède un mode présentation qui s’occupe de faire tout cela.

Le résultat en deux screens de comparaison : utilisation classique vs mode présentation

Pour activer le mode présentation ça se passe dans le menu View > Enter Presentation Mode. Et pour le quitter View > Exit Presentation Mode.

Bien évidement, assigner un raccourci clavier permettra d’être encore plus fluide pour switcher de mode.

[1] Une jolie faute d'orthographe sur le tshirt officiel de Devoxx que personne n'avait vu, mais qui a été révélée lors de la première keynote :-)

PS : Les sources github visibles dans les screens ne sont pas de moi, vous pouvez les consulter à cet endroit https://github.com/winterbe/java8-tutorial.
